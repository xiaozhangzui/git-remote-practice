# Git 远程仓库协作练习

## 任务问答

### 1. 本地仓库和远程仓库分别有什么作用？

- **本地仓库**：存储在开发者自己的电脑上，用于日常的代码编写、修改和版本管理。开发者可以随时在本地进行提交、创建分支等操作，不受网络限制。
- **远程仓库**：存储在 GitHub 等 Git 托管平台上，用于备份代码、实现多人协作和跨设备同步。远程仓库让团队成员可以共享代码、查看彼此的修改、进行代码审查等。

### 2. 为什么需要把本地提交同步到远程仓库？

本地提交只保存在个人电脑上，如果电脑出现问题（如硬盘损坏），代码就会丢失。同步到远程仓库可以实现代码备份，同时也让团队成员能够获取到最新的代码。此外，远程仓库上的提交历史也是一种协作沟通的方式，方便团队成员了解项目的进展和变化。

### 3. 远程平台直接修改文件后，本地仓库为什么需要同步远程变化？

因为远程仓库的内容已经发生了变化，本地仓库的记录还停留在之前的版本。如果不进行同步，本地和远程就会出现不一致的情况，后续再次推送时可能会产生冲突。通过 git pull 将远程变化同步到本地，可以保持本地和远程的代码一致，确保后续操作顺利进行。

### 4. 如果本地和远程同时修改了同一个文件，可能会发生什么？

可能会产生合并冲突，类似于之前分支合并时的冲突。Git 在执行 git pull 或 git push 时会检测到同一文件的不同修改，如果无法自动合并，就需要开发者手动查看冲突内容，选择保留哪个版本或综合两个版本，然后重新提交。因此，养成频繁同步的习惯可以减少冲突的发生。

### 5. 本次任务中最后一次提交为什么要只包含 README.md 的回答内容？

因为 README.md 是对整个任务的学习总结和问答记录，它本身不涉及代码功能的修改。将它作为单独的最后一次提交，可以让提交历史更加清晰：前面的提交记录了项目文件的实际操作过程，而最后的提交专门用于记录学习心得。这样查看提交历史时，能够清楚地区分实操步骤和学习总结。
